# CrackHash - Сервис распределенного взлома MD5-хэшей

Этот проект представляет собой распределенную систему для подбора MD5-хэшей, написанную на Kotlin.

## Архитектура

Система состоит из двух основных сервисов, работающих в Docker-контейнерах:

1.  **Gateway (Менеджер)**:
    *   Принимает асинхронные запросы от клиента через **GraphQL API**.
    *   Выступает в роли "менеджера": делит общую задачу по перебору комбинаций на несколько частей.
    *   Распределяет эти части между доступными `Worker`-сервисами по **gRPC**.
    *   Хранит в памяти состояние каждой задачи (`IN_PROGRESS`, `READY`, `ERROR`).
    *   Предоставляет API для проверки статуса задачи.

2.  **Worker (Рабочий)**:
    *   gRPC-сервер, который является "вычислительным ядром".
    *   Получает от `Gateway` конкретный диапазон для перебора.
    *   Циклически генерирует слова, вычисляет их MD5-хэш и сравнивает с целевым.
    *   При нахождении совпадения немедленно возвращает результат.
    *   Поддерживает отмену задачи, если другой `Worker` нашел результат раньше.

## Технологии

- **Язык**: Kotlin
- **Бэкенд**: Ktor (Gateway)
- **API**: GraphQL (`graphql-kotlin`)
- **Межсервисное взаимодействие**: gRPC
- **Контейнеризация**: Docker, Docker Compose
- **Нагрузочное тестирование**: k6

## Как запустить проект

Для запуска всей системы вам понадобится установленный Docker и Docker Compose.

1.  **Сборка и запуск контейнеров:**
    Откройте терминал в корневой директории проекта и выполните команду. Она соберет образы и запустит один `gateway` и три `worker`-а.

    ```sh
    docker-compose up --build
    ```

2.  **Остановка системы:**
    Чтобы остановить все запущенные контейнеры, выполните:
    ```sh
    docker-compose down
    ```

## Тестирование API

После запуска системы `Gateway` будет доступен по адресу `http://localhost:8080`.

Самый удобный способ протестировать API — использовать встроенный в IntelliJ IDEA HTTP Client.

1.  Откройте файл `gateway/src/test/test.http`.
2.  **Запустите первый запрос (`Start a new crack job`)**, нажав на иконку "Play" (▶). Он отправит задачу на взлом и вернет `requestId`.
3.  **Скопируйте `requestId`** из ответа.
4.  **Вставьте его во второй запрос (`Check the status of the job`)** вместо `PASTE_YOUR_REQUEST_ID_HERE`.
5.  Запускайте второй запрос, чтобы отслеживать статус задачи (`IN_PROGRESS` -> `READY`).

## Нагрузочное тестирование

Для проведения нагрузочного тестирования используется инструмент **k6**.

1.  **Установите k6** локально (инструкции на [официальном сайте](https://k6.io/docs/getting-started/installation/)).

2.  **Запустите тест:**
    Убедитесь, что система запущена через `docker-compose`. Затем выполните команду:
    ```sh
    k6 run gateway/src/test/loadtest.js
    ```

Скрипт `loadtest.js` настроен на проведение **стресс-теста**: он будет постепенно наращивать количество виртуальных пользователей (1, 2, 4, 8, 16...) и отправлять запросы на создание задач. Следите за метриками `http_req_duration` и `errors` в выводе k6, чтобы определить, при какой нагрузке система начинает деградировать.
